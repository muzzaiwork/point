# Admin 포인트 관리 시스템 기획서

## 개요

포인트 Admin은 운영자가 사용자 포인트 현황을 조회·분석할 수 있는 내부 관리 도구입니다.  
Spring Boot + Thymeleaf 기반의 서버사이드 렌더링 웹 UI로 제공되며, 별도 인증 없이 `/admin` 경로로 접근합니다.

---

## 공통 레이아웃

모든 Admin 페이지는 동일한 헤더 + 네비게이션 구조를 공유합니다.

```
┌─────────────────────────────────────────────────────────────┐
│  🛠 Point Admin   사용자 계정  포인트 적립 내역  포인트 사용 내역  통계  │
└─────────────────────────────────────────────────────────────┘
```

| 메뉴 | URL | 설명 |
|------|-----|------|
| 사용자 계정 | `/admin/accounts` | 사용자별 포인트 잔액 현황 |
| 포인트 적립 내역 | `/admin/points` | 포인트 단건 적립/취소/만료 내역 |
| 포인트 사용 내역 | `/admin/orders` | 주문 기반 포인트 사용/취소 내역 |
| 통계 | `/admin/stats` | 일별/월별/연도별 집계 통계 |

## 1. 사용자 계정 (`/admin/accounts`)

### 화면 구성

```
┌──────────────────────────────────────────────────────┐
│  [사용자 ID 입력]  [이름 입력]  [조회]  [초기화]          │
└──────────────────────────────────────────────────────┘
총 N건 (페이지 1 / M)

┌────┬──────────┬──────┬──────────────────┬──────────────────┬──────────────────┬──────────┬──────────┐
│ ID │ userId   │ 이름  │ 총 적립 (유료/무료) │ 총 사용 (유료/무료) │ 잔여 포인트 (유료/무료) │ 1회 적립 한도 │ 보유 한도 │
└────┴──────────┴──────┴──────────────────┴──────────────────┴──────────────────┴──────────┴──────────┘
```

### 검색 조건

| 필드 | 타입 | 설명 |
|------|------|------|
| 사용자 ID | 텍스트 | userId 부분 일치 검색 |
| 이름 | 텍스트 | 이름 부분 일치 검색 |

### 테이블 컬럼

| 컬럼 | 설명 |
|------|------|
| ID | DB 내부 ID |
| userId | 사용자 식별자 |
| 이름 | 사용자 이름 |
| 총 적립 (유료/무료) | 누적 적립 포인트 합계 (유료/무료 분리 표시). 클릭 시 해당 사용자의 포인트 적립 내역으로 이동 |
| 총 사용 (유료/무료) | 누적 사용 포인트 합계 (유료/무료 분리 표시). 클릭 시 해당 사용자의 포인트 사용 내역으로 이동 |
| 잔여 포인트 (유료/무료) | 현재 보유 포인트 (유료/무료 분리 표시) |
| 1회 적립 한도 | 1회 최대 적립 가능 금액 |
| 보유 한도 | 최대 보유 가능 포인트 |

### 페이지네이션

- 페이지당 20건
- 이전/다음 + 페이지 번호 표시 (현재 페이지 기준 ±2)
- 등록일시 내림차순 정렬

## 2. 포인트 적립 내역 (`/admin/points`)

### 화면 구성

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  [사용자 ID]  [취소 여부 ▼]  [포인트 타입 ▼]  [적립 출처 ▼]                      │
│  [적립일자 시작]  [적립일자 종료]  [조회]  [초기화]                                │
└──────────────────────────────────────────────────────────────────────────────┘
총 N건 (페이지 1 / M)

┌──────────┬──────────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────────┬──────────┬──────────┬──────┬──────────┐
│ pointKey │ 사용자 ID │ 타입  │ 출처  │ 만료  │ 취소  │ 적립  │ 잔여  │ 사용  │사용취소│ 만료금액 │rootPointKey│ 원천 pointKey │ 재지급 pointKey │ 만료일 │ 적립일시 │
└──────────┴──────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────────┴──────────┴──────────┴──────┴──────────┘
```

### 검색 조건

| 필드 | 타입 | 옵션 |
|------|------|------|
| 사용자 ID | 텍스트 | - |
| 취소 여부 | 셀렉트 | 전체 / 취소 |
| 포인트 타입 | 셀렉트 | 전체 / 무료(FREE) / 유료(PAID) |
| 적립 출처 | 셀렉트 | 전체 / 적립(ACCUMULATION) / 수기지급(MANUAL) / 만료후취소재적립(AUTO_RESTORED) |
| 적립일자 시작 | 날짜 | - |
| 적립일자 종료 | 날짜 | - |

### 테이블 컬럼

| 컬럼 | 설명 |
|------|------|
| pointKey | 포인트 고유 키. 클릭 시 해당 포인트의 이벤트 이력 모달 팝업 |
| 사용자 ID | 포인트 소유자 |
| 타입 | 무료(파란색 뱃지) / 유료(노란색 뱃지) |
| 출처 | 적립(초록) / 수기지급(보라) / 만료후취소재적립(분홍) |
| 만료 여부 | 만료된 경우 빨간 뱃지 표시 |
| 취소 여부 | 취소된 경우 빨간 뱃지 표시 |
| 적립 금액 | 최초 적립된 금액 |
| 잔여 금액 | 현재 남은 금액 |
| 사용 금액 | 사용된 금액 합계 |
| 사용취소 금액 | 사용 취소된 금액 합계 |
| 만료 금액 | 만료된 금액 |
| rootPointKey | 재발급 체인의 최상위 포인트 키. 클릭 시 연관 포인트 내역 모달 팝업 |
| 원천 pointKey | 만료 후 재지급된 경우 원천 포인트 키 |
| 재지급 pointKey | 이 포인트에서 재지급된 포인트 키 |
| 만료일 | 포인트 만료 예정일 |
| 적립일시 | 포인트 적립 일시 |

### 모달 팝업 1 — 포인트 이벤트 이력

pointKey 클릭 시 해당 포인트에 발생한 모든 이벤트를 시계열로 표시합니다.

```
┌─────────────────────────────────────────────────────┐
│  포인트 이벤트 이력  [pointKey]                    ✕  │
├─────┬──────────┬──────────┬──────────┬──────┬──────┤
│  #  │ 이벤트 타입 │   금액    │  주문번호  │취소 ID│취소금액│
├─────┼──────────┼──────────┼──────────┼──────┼──────┤
│  1  │  적립     │ +50,000  │    -     │  -   │  -   │
│  2  │  사용     │ -30,000  │ ORD-001  │  -   │  -   │
│  3  │  사용취소  │ +30,000  │ ORD-001  │  1   │30,000│
├─────┴──────────┼──────────┴──────────┴──────┴──────┤
│   잔여 합계      │          +50,000                  │
└────────────────┴───────────────────────────────────┘
```

**이벤트 타입 색상 코드**

| 이벤트 타입 | 한글명 | 금액 부호 | 색상 |
|------------|--------|----------|------|
| ACCUMULATE | 적립 | + | 초록 |
| USE | 사용 | - | 파랑 |
| ACCUMULATE_CANCEL | 적립취소 | - | 빨강 |
| USE_CANCEL | 사용취소 | + | 주황 |
| EXPIRE | 만료 | - | 보라 |
| EXPIRED_CANCEL_RESTORE | 만료후취소재적립 | + | 분홍 |

### 모달 팝업 2 — 연관 포인트 내역 (rootPointKey)

rootPointKey 클릭 시 동일 체인에 속한 모든 포인트(원천 + 재지급)를 표시합니다.

```
┌──────────────────────────────────────────────────────────────────────────┐
│  연관 포인트 내역  [rootPointKey]                                       ✕  │
├──────────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┤
│ pointKey │ 타입  │ 출처  │ 상태  │ 적립  │ 잔여  │ 사용  │ 만료  │원천 key│ 만료일 │
└──────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

## 3. 포인트 사용 내역 (`/admin/orders`)

### 화면 구성

```
┌──────────────────────────────────────────────────────────────────────────┐
│  [사용자 ID]  [주문번호]  [주문 타입 ▼]  [시작일]  [종료일]  [조회]  [초기화]  │
└──────────────────────────────────────────────────────────────────────────┘
총 N건 (페이지 1 / M)

┌──────────┬──────────┬──────────┬──────┬──────────┬──────────┬──────────┬──────────┐
│ 주문 ID   │ 사용자 ID │  주문번호  │ 타입  │ 사용 금액  │ 취소 금액  │  주문일시  │  취소일시  │
└──────────┴──────────┴──────────┴──────┴──────────┴──────────┴──────────┴──────────┘
```

### 검색 조건

| 필드 | 타입 | 옵션 |
|------|------|------|
| 사용자 ID | 텍스트 | - |
| 주문번호 | 텍스트 | - |
| 주문 타입 | 셀렉트 | 전체 / USE / USE_CANCEL |
| 시작일 | 날짜 | - |
| 종료일 | 날짜 | - |

### 테이블 컬럼

| 컬럼 | 설명 |
|------|------|
| 주문 ID | DB 내부 ID |
| 사용자 ID | 포인트 사용자 |
| 주문번호 | 외부 주문 식별자 |
| 타입 | USE(사용) / USE_CANCEL(취소) |
| 사용 금액 | 해당 주문에서 사용된 포인트 총액. 클릭 시 포인트별 사용 상세 모달 팝업 |
| 취소 금액 | 취소된 포인트 총액. 클릭 시 취소 이력 모달 팝업 |
| 주문일시 | 주문 생성 일시 |
| 취소일시 | 취소 처리 일시 |

## 4. 통계 (`/admin/stats`)

### 화면 구성

```
┌──────────────────────────────────────────────────────┐
│  [일별]  [월별]  [연도별]  ← 탭 전환                    │
├──────────────────────────────────────────────────────┤
│  (일별)   [시작일 ──────]  [종료일 ──────]  [조회] [초기화] │
│  (월별)   [시작 월 ──────]  [종료 월 ──────]  [조회] [초기화] │
│  (연도별) [시작 연도 ────]  [종료 연도 ────]  [조회] [초기화] │
└──────────────────────────────────────────────────────┘

┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┬──────────────────┐
│  날짜/연월/연도 │   적립    │  적립취소  │   사용    │  사용취소  │   만료    │  만료후취소재지급  │
├──────────┼──────────┼──────────┼──────────┼──────────┼──────────┼──────────────────┤
│ 2025-12-31│  63,000  │   3,000  │  44,000  │  17,000  │  13,000  │      14,000      │
│    ...   │   ...    │   ...    │   ...    │   ...    │   ...    │       ...        │
├──────────┼──────────┼──────────┼──────────┼──────────┼──────────┼──────────────────┤
│   합계    │17,385,000│  825,000 │12,241,000│ 4,636,000│ 3,620,000│    3,940,000     │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┴──────────────────┘
```

### 단위별 검색 조건

| 단위 | 검색 필드 | 기본값 |
|------|----------|--------|
| 일별 | 시작일 ~ 종료일 (`type="date"`) | 2025-01-01 ~ 2025-12-31 |
| 월별 | 시작 월 ~ 종료 월 (`type="month"`) | 2025-01 ~ 2025-12 |
| 연도별 | 시작 연도 ~ 종료 연도 (`type="number"`) | 2025 ~ 2025 |

### 컬럼 색상 코드

| 컬럼 | 헤더 색상 | 의미 |
|------|----------|------|
| 적립 | 초록 (`#2f6b3e`) | 신규 포인트 적립 |
| 적립취소 | 빨강 (`#7b3030`) | 적립 취소 |
| 사용 | 파랑 (`#1a4a7a`) | 포인트 사용 |
| 사용취소 | 주황 (`#7a5a1a`) | 사용 취소 |
| 만료 | 보라 (`#4a2a7a`) | 포인트 만료 |
| 만료후취소재지급 | 분홍 (`#7a1a4a`) | 만료 후 취소 시 재지급 |

### 특이사항

- 조회 범위 내 데이터가 없는 날짜/월/연도도 0으로 채워서 표시
- 합계 행은 항상 테이블 하단에 고정 표시
- 결과는 날짜 내림차순(최신 → 과거) 정렬

## API 엔드포인트 (내부 AJAX)

Admin 페이지 내 모달 팝업은 아래 API를 비동기 호출합니다.

| 메서드 | URL | 파라미터 | 설명 |
|--------|-----|----------|------|
| GET | `/admin/point-events` | `pointKey` | 특정 포인트의 이벤트 이력 조회 |
| GET | `/admin/points-by-root-key` | `rootPointKey` | 동일 체인의 연관 포인트 목록 조회 |
| GET | `/admin/order-cancels` | `orderId` | 주문의 취소 이력 및 포인트별 취소 상세 조회 |
| GET | `/admin/order-use-details` | `orderId` | 주문의 포인트별 사용 상세 조회 |

## 기술 스택

| 항목 | 내용 |
|------|------|
| 백엔드 | Spring Boot, Spring Data JPA |
| 뷰 엔진 | Thymeleaf |
| DB | H2 (파일 모드, `data/billing.mv.db`) |
| 집계 쿼리 | JPQL + H2 `FORMATDATETIME`, `YEAR` 함수 |
| 스타일 | 인라인 CSS (외부 라이브러리 없음) |
| 비동기 | Fetch API (모달 팝업) |
