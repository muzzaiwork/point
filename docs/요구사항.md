3. 필요 기능 : 적립, 적립취소, 사용, 사용취소
    1. 적립
        1. 1회 적립가능 포인트는 1포인트 이상, 10만포인트 이하로 가능하며 1회 최대 적립가능 포인트는 하드코딩이 아닌 방법으로 제어할수 있어야 한다.
           ```java
           // PointService.java
           @Value("${point.accumulation.max-limit}")
           private long maxSystemAccumulationLimit;

           public String accumulate(...) {
               // 시스템 공통 적립 상한 검증
               if (amount > maxSystemAccumulationLimit) {
                   throw new BusinessException(ResultCode.BAD_REQUEST,
                       "시스템 적립 상한(" + maxSystemAccumulationLimit + ")을 초과하였습니다.");
               }
               // ...
           }
           ```
           ```yaml
           # application.yml
           point:
             accumulation:
               max-limit: 100000
           ```
        2. 개인별로 보유 가능한 무료포인트의 최대금액 제한이 존재하며, 하드코딩이 아닌 별도의 방법으로 변경할 수 있어야 한다.
           ```java
           // UserAccount.java
           public void accumulatePoint(Long amount, PointType type) {
               if (amount > this.maxAccumulationPoint) {
                   throw new BusinessException(ResultCode.ACCUMULATION_LIMIT_EXCEEDED,
                       "1회 적립 가능 한도(" + this.maxAccumulationPoint + ")를 초과하였습니다.");
               }
               if (this.remainingPoint + amount > this.maxRetentionPoint) {
                   throw new BusinessException(ResultCode.RETENTION_LIMIT_EXCEEDED,
                       "개인별 최대 보유 가능 포인트(" + this.maxRetentionPoint + ")를 초과할 수 없습니다.");
               }
               this.remainingPoint += amount;
               // ...
           }
           ```
        3. 특정 시점에 적립된 포인트는 1원단위까지 어떤 주문에서 사용되었는지 추적할수 있어야 한다.
           ```java
           // PointService.java (use 메서드)
           for (Point point : availablePoints) {
               if (remainingToUse <= 0) break;

               long canUseFromThis = Math.min(point.getRemainingPoint(), remainingToUse);
               point.use(canUseFromThis);
               remainingToUse -= canUseFromThis;

               PointEvent useEvent = PointEvent.builder()
                       .order(order)       // 어떤 주문인지 (Order)
                       .point(point)       // 어떤 적립 건인지
                       .pointEventType(PointEventType.USE)
                       .amount(canUseFromThis) // 사용 금액 (1원 단위)
                       .build();
               pointEventRepository.save(useEvent);
           }
           ```
        4. 포인트 적립은 관리자가 수기로 지급할 수 있으며, 수기지급한 포인트는 다른 적립과 구분되어 식별할 수 있어야 한다.
           ```java
           // PointSourceType.java
           public enum PointSourceType {
               ACCUMULATION,   // 일반 적립
               MANUAL,         // 관리자 수기 지급
               AUTO_RESTORED   // 만료 후 사용취소 재적립
           }

           // Point.java
           @Enumerated(EnumType.STRING)
           @Column(nullable = false)
           private PointSourceType pointSourceType; // MANUAL이면 수기 지급 건으로 식별
           ```
        5. 모든 포인트는 만료일이 존재하며, 별도의 기간 제한 없이 부여할 수 있다. (기본 2999-12-31)
           ```java
           // PointService.java
           private LocalDateTime resolveExpiryDate(Integer expiryDays) {
               if (expiryDays != null) {
                   return LocalDateTime.now().plusDays(expiryDays)
                           .withHour(23).withMinute(59).withSecond(59);
               }
               return LocalDateTime.of(2999, 12, 31, 23, 59, 59);
           }
           ```
    2. 적립 취소
        1. 특정 적립행위에서 적립한 금액만큼 취소 가능하며, 적립한 금액중 일부가 사용된 경우라면 적립 취소 될 수 없다.
           ```java
           // Point.java
           public void cancel() {
               if (this.isCancelled) {
                   throw new BusinessException(ResultCode.ALREADY_CANCELLED);
               }
               if (!this.accumulatedPoint.equals(this.remainingPoint)) {
                   throw new BusinessException(ResultCode.ALREADY_USED);
               }
               this.isCancelled = true;
               this.remainingPoint = 0L;
           }
           ```
    3. 사용
        1. 주문시에만 포인트를 사용할 수 있다고 가정한다.
           ```java
           // PointDto.java
           public static class UseRequest {
               @NotBlank(message = "주문 번호는 필수입니다.")
               private String orderNo;
               // ...
           }
           ```
        2. 포인트 사용시에는 주문번호를 함께 기록하여 어떤 주문에서 얼마의 포인트를 사용했는지 식별할 수 있어야 한다.
           ```java
           // Order.java
           @Column(nullable = false, unique = true)
           private String orderNo;     // 주문 번호 (식별자로 사용)
           private Long orderedPoint;  // 사용 총액
           ```
        3. 포인트 사용시에는 관리자가 수기 지급한 포인트가 우선 사용되어야 하며, 만료일이 짧게 남은 순서로 사용해야 한다.
           ```java
           // PointRepository.java
           @Query("SELECT p FROM Point p " +
                  "WHERE p.userId = :userId AND p.remainingPoint > 0 " +
                  "AND p.isExpired = false AND p.isCancelled = false " +
                  "ORDER BY CASE WHEN p.pointSourceType = 'MANUAL' THEN 1 ELSE 0 END DESC, p.expiryDateTime ASC")
           List<Point> findAvailablePoints(@Param("userId") String userId);
           ```
    4. 사용 취소
        1. 사용한 금액중 전체 또는 일부를 사용취소 할수 있다.
           ```java
           // PointService.java (cancelUsage 메서드)
           public void cancelUsage(String orderNo, Long cancelAmount) {
               // ...
               order.cancel(cancelAmount); // 취소 금액 누적 및 상태 갱신 (PARTIAL_CANCEL / TOTAL_CANCEL)
               user.cancelUsage(canCancelFromThis, point.getType()); // 사용자 잔액 복구
               // ...
           }
           ```
           ```java
           // Order.java
           public void cancel(Long cancelAmount) {
               if (this.canceledPoint + cancelAmount > this.orderedPoint) {
                   throw new BusinessException(ResultCode.CANCEL_AMOUNT_EXCEEDED);
               }
               this.canceledPoint += cancelAmount;
               this.type = this.canceledPoint.equals(this.orderedPoint)
                       ? OrderType.TOTAL_CANCEL : OrderType.PARTIAL_CANCEL;
           }
           ```
        2. 사용취소 시점에 이미 만료된 포인를 사용취소 해야 한다면 그 금액만큼 신규적립 처리 한다.
           ```java
           // PointService.java (cancelUsage 메서드)
           if (point.isExpired()) {
               // 만료된 경우: AUTO_RESTORED 타입 신규 적립 처리
               //   - originPointKey / rootPointKey 는 원본에서 상속
               //   - 만료일: 2999-12-31 (사실상 무기한)
               //   - PointEvent(EXPIRED_CANCEL_RESTORE) 기록
               user.cancelUsage(canCancelFromThis, point.getType());
               doAccumulate(order.getUserId(), canCancelFromThis,
                       PointSourceType.AUTO_RESTORED, point.getType(),
                       LocalDateTime.of(2999, 12, 31, 23, 59, 59),
                       null, point.getPointKey(), point.getRootPointKey(), orderCancel);
           } else {
               // 유효한 경우: 기존 적립 건 잔액 복구
               //   - PointEvent(USE_CANCEL) 기록
               point.restore(canCancelFromThis);
               user.cancelUsage(canCancelFromThis, point.getType());
               // ...
           }
           ```
4. 예시
    1. 1000원 적립한다 (총 잔액 0 -> 1000 원)
        1. pointKey : A 로 할당
    2. 500원 적립한다 (총 잔액 1000 -> 1500 원)
        1. pointKey : B 로 할당
    3. 주문번호 A1234 에서 1200원 사용한다 (총 잔액 1500 -> 300 원)
        1. 식별값(orderNo) : A1234
        2. A 적립에서 1000원 사용함 -> A의 사용가능 잔액은 1000 -> 0
        3. B 적립에서 200원 사용함 -> B의 사용가능 잔액은 500 -> 300
    4. A의 적립이 만료되었다
    5. A1234의 사용금액 1200원 중 1100원을 부분 사용취소 한다 (총 잔액 300 -> 1400 원)
        1. 1200원은 A와 B에서 사용되었다.
        2. 그래서, 사용취소 하면 A의 사용가능 잔액이 0 -> 1000원 되어야 하지만, A는 이미 만료일이 지났기 때문에 새로운 pointKey로 1000원이 신규적립 되어야 한다.
        3. B는 만료되지 않았기 때문에 사용가능 잔액은 300 -> 400원이 된다.
        4. A1234는 이제 1200원 사용금액중 100원을 부분취소 할 수 있다.
